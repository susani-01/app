## Data Quality Issues

- 10,845 classification rows have no `qtyCalcCtyclcd`; they represent hierarchy nodes, not billable items.
- 28,945 item codes exist in classification but have no matching price record.
- 117 distinct unit strings are not yet mapped to a canonical form and need review during ETL.
- Equivalent units appear under multiple spellings (for example `m`/`M`, `㎡`/`M2`/`m2`).

## Design Implications for Phase 2+

- Treat `construction_classification.jsonl` as the master catalog and `std_market_price.jsonl` as optional enrichment linked by `qtyCalcCtyclcd`.
- Model classification hierarchy nodes separately from leaf-level quantity items.
- Only 14.6% of classified item codes currently have a price; the API must expose `price: null` explicitly.
- Normalize units during ETL before persisting data or serving search/filter results.
- Use a left join from items to prices so items without prices remain searchable.
- Index item name (`qtyCalcCtyclNm`) and classification filters for the search API.
